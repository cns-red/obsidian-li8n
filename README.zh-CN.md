# Internationalization for Markdown

**在单个笔记中编写多种语言，全局切换可见语言。**

- [English](./README.md)
- [BiliBili Video](https://www.bilibili.com/video/BV1RrcrzbEox)

---

## 功能简介

mi18n 让你把同一篇笔记的所有语言版本放在一个文件里，然后只显示你当前需要的语言。通过 Ribbon、状态栏或命令面板全局切换语言 —— 所有已打开的笔记在阅读模式、实时预览和源码模式下同步更新。

## 快速上手

用 `lang` 块包裹各语言内容：

```markdown
:::lang en
Hello, world!
:::

:::lang zh-CN
你好，世界！
:::
```

点击 Ribbon 图标或状态栏徽标选择语言。只有对应语言的块可见 —— 其余内容在阅读模式和编辑器中均被隐藏。

---

## 语法样式
> 已集成到右键菜单，可便捷插入无需手动编写语法

四种语法完全等价，可在同一笔记中混合使用：

| 样式 | 开始标记 | 结束标记 | 阅读模式可见 |
|---|---|---|---|
| Fenced div（默认） | `:::lang zh-CN` | `:::` | 是 |
| Hexo 标签 | `{% lang zh-CN %}` | `{% endlang %}` | 是 |
| Markdown 注释 | `[//]: # (lang zh-CN)` | `[//]: # (endlang)` | **否** |
| Obsidian 注释 | `%% lang zh-CN %%` | `%% endlang %%` | **否**（实时预览中也不可见） |

**多语言块** —— `:::lang en zh-CN` 在任意一种语言激活时均显示。

**规则：**
- 开始和结束标记必须独占一行，行首不能有空格。
- 语言代码匹配不区分大小写（`zh-CN` 与 `zh-cn` 视为相同）。
- 语言块之外的内容在所有语言下均可见。
- 未关闭的块延伸至文件末尾。

**拓展：** 未来将更新适配 Hexo 等基于 Markdown 文件的博客系统多语言插件同步使用。如 `hexo-mi18n`，对该项目计划感兴趣可以联系我。

## Frontmatter

mi18n 会自动保持这些字段同步 —— 通常无需手动填写：

```yaml
---
lang: [en, zh-CN, ja]   # 笔记中存在的语言（保存时自动同步）
lang_view: zh-CN         # 每次打开此笔记时锁定为指定语言
lang_ignore: true        # 单篇排除, 对此笔记完全禁用 mi18n
title: "我的文章"        # 默认语言的内联标题（替代文件名显示）
title_en: "My Article"  # 各语言独立标题（格式：title_<语言代码>）
---
```

开启**从 Frontmatter 覆盖笔记标题**（设置 → 界面）后，mi18n 会用 `title`（或对应的 `title_<语言代码>` 变体）替代文件名显示为阅读模式顶部的内联标题。`title` 用于默认语言，同时也是未配置独立变体的语言的回退值。

点击内联标题会临时还原为真实文件名，可正常编辑重命名；完成后 mi18n 自动重新渲染翻译标题。

## UI 控件

### Ribbon 按钮
左侧边栏图标。打开语言选择菜单，包含所有已配置语言以及**显示全部语言**选项。可在设置中隐藏。

### 状态栏徽标
右下角显示当前激活的语言代码。当前笔记缺少某些语言的翻译时，会出现警告点。点击打开完整语言菜单。

徽标旁有两个图标按钮：
- **对比** —— 打开语言对比对话框
- **语言** —— 效果同点击徽标

### 阅读模式语言栏
在每篇多语言笔记的阅读视图顶部，自动注入浮动语言切换栏。点击任意语言按钮切换。包含 **ALL** 选项可同时显示所有块。

### 大纲面板集成
大纲面板打开时，顶部会注入语言切换控件。非激活语言对应的标题会被自动隐藏，保持大纲整洁。

## 编辑器右键菜单

在编辑器任意位置右键 → **Multilingual** 子菜单：

| 操作 | 说明 |
|---|---|
| **Wrap（包裹）** | 用当前激活语言的 lang 块包裹选中文本 |
| **Copy（复制）→** | 将指定语言的完整内容复制到剪贴板 |
| **Paste as…（粘贴为）→** | 将剪贴板内容作为指定语言的 lang 块粘贴 |
| **Delete（删除）→** | 从笔记中删除指定语言的所有块 |
| **Manual insert（手动插入）→** | 在光标处插入指定语言的空 lang 块 |
| **Smart insert（智能插入）** | 自动检测缺失语言并插入下一个所需块 |
| **Smart AI translate（AI 翻译）** | 打开 AI 翻译对话框 |

悬停在 **复制 / 粘贴为 / 删除 / 手动插入** 上，会弹出语言选择器。**粘贴为**和**手动插入**中已存在的语言会显示为禁用状态。

![menu](/examples/images/fcb50e9f-a315-411c-b767-bc5b02ecf9ab.png)

## 文件管理器右键菜单

在文件管理器中右键任意 Markdown 文件 → **Multilingual → Export（导出）→ \<语言\>**

将该语言的内容（已去除 lang 标记）导出为独立的 `.md` 文件。若指定语言的块不存在，将导出块外的共享内容并给出提示。

## 命令（命令面板）

| 命令 | 说明 |
|---|---|
| `Switch language: <名称>` | 切换到指定语言（每种配置语言各一条命令） |
| `Switch language: Show all languages` | 同时显示所有语言块 |
| `Cycle to next language` | 按顺序轮换到下一种语言 |
| `Insert language block` | 在光标处插入空 lang 块 |
| `Smart insert language block` | 智能插入下一个缺失语言块 |
| `Wrap selection in language block` | 用 lang 块包裹选中文本 |
| `Smart AI translate` | 打开 AI 翻译对话框 |
| `Insert multilingual block template (all languages)` | 一次性插入所有已配置语言的空块 |

## 语言对比

打开同一笔记不同语言版本的并排分屏，支持同步滚动。

**打开方式：** 点击状态栏的对比图标 → 选择两种或多种语言 → 点击**应用分屏视图**。每个分屏独立锁定所显示的语言。点击**返回普通模式**关闭对比会话。

## AI 翻译

**智能 AI 翻译**对话框包含：

| 元素 | 说明 |
|---|---|
| 源语言面板 | 所选源语言内容的渲染 Markdown 预览 |
| 翻译面板 | AI 逐 token 实时流式输出 |
| 编辑切换 | 在渲染预览与纯文本编辑之间切换翻译面板 |
| 重新生成 | 使用相同设置重新运行翻译 |
| 插入 | 将结果作为新的 lang 块追加到笔记中 |

关闭对话框或点击**取消**会立即中止 API 请求 —— 不会浪费任何 token。

支持任意兼容 OpenAI 的 API：OpenAI、Ollama、OpenRouter、SiliconFlow 等。在 **设置 → AI 翻译** 中配置。

## 设置

打开 **设置 → mi18n**：

### 语言库
- **激活语言** —— 当前所有笔记的可见语言
- **默认语言** —— 无 lang 标记笔记的默认语言（切换到其他语言会让此类笔记不可见）
- **已配置语言** —— 添加 / 重命名 / 删除语言条目；代码必须与标记中的完全一致

### 界面
- **从 Frontmatter 覆盖笔记标题** —— 用 `title` / `title_<语言代码>` 字段的值替代文件名显示为内联标题；点击标题可临时还原文件名进行重命名
- **在编辑器中隐藏其他语言** —— 在实时预览 / 源码模式中将非激活块折叠为细条占位符
- **在阅读模式显示语言切换栏** —— 控制阅读模式顶部的语言按钮栏
- **显示 Ribbon 按钮** —— 控制左侧边栏图标
- **显示状态栏指示器** —— 控制右下角状态栏徽标

### AI 翻译
API Base URL · API Key · 模型（如 `gpt-4o-mini`）· 系统提示词

### 作用范围
限制 mi18n 在特定 vault 文件夹内生效：
- **工作目录** —— 插件仅在列出的文件夹内激活（留空 = 所有文件）
- **排除目录** —— 插件在这些文件夹内完全禁用（优先级高于工作目录）

---

## 无 lang 标记的笔记

没有任何 lang 标记的笔记被视为完全用**默认语言**编写。切换到其他语言时，整篇笔记会不可见 —— 这是有意为之，因为该笔记没有对应语言的翻译。
